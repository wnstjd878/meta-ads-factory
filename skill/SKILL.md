# 메타 광고 운영 스킬

메타 광고 계정을 내 판단 기준표로 자동 운영한다. 명령어 3개로 돈다.

| 명령어 | 하는 일 |
|---|---|
| `/판정` | 계정 성과를 기준표대로 판정 (끄기 확정/후보 · 유지 · 증액 후보) |
| `/소재생성` | 리뷰·커뮤니티에서 고객 언어를 뽑아 카피 20개 + 소재 설계 |
| `/채점` | 내 소재나 광고를 점수화하고 약점·개선안 제시 |

메타 광고 계정이 연결(MCP)돼 있으면 데이터를 자동으로 가져온다. 연결이
없으면 사용자에게 "광고 관리자에서 내보낸 파일이나 화면 캡처를 달라"고
요청해서 똑같이 분석한다.

핵심 원칙: 이 스킬은 읽기·분석·생성만 한다. 캠페인 생성/수정/삭제 같은
실제 집행 변경은 하지 않는다. 변경이 필요하면 사용자가 광고 관리자에서
직접 한다. (자동 집행은 돈이 걸리고 소재는 사람이 검수해야 하므로.)

## 판단 기준표

기준표는 2블록이다. 블록 A(공통 룰)는 그대로 쓰고, 블록 B(내 계정)는
사용자가 자기 숫자로 채운다. `/판정`은 이 기준표를 근거로 판단한다.

### 블록 A · 공통 룰 (고정)

0. 모든 금액은 광고 계정 통화 기준으로만 판단한다. 환율 환산 금지.
   BEP와 성과 데이터의 통화가 다르면 판정을 멈추고 사용자에게 알린다.
   (달러 계정이면 BEP도 달러로 받는다.)

끄기 판정

1. **구간으로 먼저 나눈다. 예외 없이 이 표를 따른다.**

   **구간을 나누는 숫자는 지난 14일 CPA 하나다.** 최근 5일 같은 짧은 구간
   숫자로 그룹을 바꾸지 않는다. 짧은 구간은 규칙 2·3의 추세 판단에만 쓴다.

   | 14일 결과당 비용(CPA) | 그룹 |
   |---|---|
   | CPA ≤ BEP | 증액 후보 (규칙 6으로) |
   | BEP < CPA ≤ BEP×2 | **유지** (소재 교체는 검토, 끄지는 않음) |
   | CPA > BEP×2 | 끄기 후보 |

   BEP와 BEP×2 사이는 전부 "유지"다. BEP를 조금 넘었다는 이유로 끄기 후보에
   넣지 말 것. 같은 구간의 캠페인이 서로 다른 그룹으로 가면 그건 오판이다.
   예산이 빠듯해 경계를 ×1.3~1.5로 낮추려면, 판정 시작 전에 사용자에게
   "×2 대신 ×1.5를 쓸까요?"라고 묻고 답을 받은 뒤 적용한다. 혼자 낮추지 않는다.

   14일은 유지 구간인데 최근 5일이 BEP×2를 넘겼다면, 그룹은 유지로 두고
   "며칠 더 오르면 끄기 후보로 넘어감"이라고 경고를 붙인다. 그룹을 옮기지 않는다.

2. 끄기 후보라도 일자별 CPA가 내려가는 추세면 보류 (시간차 기여형).
3. 끄기 확정은 **끄기 후보 안에서만** 고른다. 하루 데이터로 확정 금지.
   최근 2·3·4·5일 구간을 각각 보고, 전부에서 하위인 것만 "끄기 확정".
   아니면 "끄기 후보"까지만. 구간별 순위를 근거로 함께 적는다.
4. CTR 높은데 전환 0 = 어그로 소재 → 끄기 후보 (구간과 무관하게).
5. 14일 지출이 BEP보다 적으면(결과가 나올 규모가 안 됨) 판정 보류.
   결과 1건짜리 CPA도 제외(표본 부족·픽셀 오류 가능).
   **이 기준선은 BEP다. BEP×2 같은 다른 값으로 올리지 말 것.**

늘리기 판정
6. CPA가 BEP 아래 + 반응 지표(공감/공유/저장/댓글) 우세 → 증액 1순위.
7. 증액은 기존 예산을 올리지 말고 "복제 증액"으로만 제안.
8. 빈도 2~3 이상이어도 CPA가 계속 내려가면 교체 말고 증액.

예산
9. 1일 예산 상한 = **BEP × 7** (7일에 결과 50건이 목표). 사용자가 블록 B에
   직접 다른 값을 적어줬을 때만 그 값을 쓴다.
10. 계정에 이미 걸린 예산은 참고로 보고만 하고, 판정 기준선으로 삼지 않는다.
    계정 예산과 BEP×7이 다르면 둘 다 적고 차이를 한 줄로 짚는다.
    (예: "계정 상한 $20, 기준 상한 $91. 계정이 7일 50건 속도에 못 미침.")

구조
9. 새 소재는 항상 111(1캠페인 1광고세트 1광고)로 시작.
10. 같은 메시지(소구)의 생존 소재가 2개 이상이면 ABO 승격 제안
    (예산 = 합계 × 1.5~3배).
11. ABO는 같은 메시지끼리만. 다른 메시지는 CBO 단계에서만 함께.
12. 잘 나가던 소재가 갑자기 죽으면 = 모수 소진 → 새 소구 제작 제안.

데이터 한계 (정직하게 지킬 것)
- 1일/7일/28일 "기여 설정별" 결과당 비용은 지금 도구로 잘 안 나온다.
  대신 "일자별(하루 단위)" 데이터는 나온다 → 규칙 2·3은 일자별로 적용.
- 데이터가 없으면 지어내지 말고 "이 지표는 못 가져옴"이라고 말한다.

안전 (절대 규칙)
- 켜기/끄기/예산 변경/삭제는 실행하지 않는다. "제안"만 한다.
- 판정마다 어떤 규칙 몇 번에 걸렸는지 근거 숫자와 함께 쓴다.

### 블록 B · 내 계정 (실행할 때 물어봐서 채움)

이 값들은 사람마다 다르다. `/판정`을 실행할 때 아래 4가지가 정해져
있지 않으면, 판정하기 전에 사용자에게 먼저 물어보고 답을 받은 뒤 판정한다.
파일을 고치지 않아도 대화만으로 자기 계정에 맞춰진다.

물어볼 4가지:
- 업종: 커머스 / 리드젠
- 결과: 커머스=구매 / 리드젠=DB(상담 신청·등록 완료)
- 계정 통화: 광고 계정 통화 그대로 (USD 계정이면 BEP도 USD로 받는다)
- BEP CPA: 결과 1건 손익분기 비용 (1일 예산 상한도. 없으면 BEP×7로 제안)

서비스가 여러 개면 서비스별 BEP를 받아, 캠페인 이름의 구분 코드로 어느
기준을 적용할지 매칭한다. 코드 없는 캠페인은 잠정 판정만 하고 끄기/증액
확정에서 제외 → 코드 지정을 요청한다.

매번 묻는 게 싫은 사용자는 이 블록에 값을 직접 적어두면 그때부턴 안 묻는다.
(예: 업종=리드젠 / 결과=DB / 통화=USD / BEP=$14 / 1일 상한=$21)

## 명령어 정의

### /판정
기준표대로 지난 14일 광고를 판정한다. (계정 연결 없으면 사용자가 준
데이터로.)

판정 전에 반드시 할 것:
- 블록 B의 4가지(업종·결과·통화·BEP)가 없으면 먼저 묻는다.
- 기준선 세 개를 화면에 먼저 적는다: `BEP=$X / 끄기 후보선=$X×2 / 1일 상한=$X×7`
- 그다음 캠페인마다 CPA가 어느 구간인지 규칙 1 표에 대입한다.

출력:
- 끄기 확정 / 끄기 후보(보류 사유) / 유지 / 증액 후보 4그룹으로 분류
- 캠페인마다 `CPA $20.45 = BEP의 1.57배 → 유지 구간` 처럼 배수를 적는다.
  배수를 안 적으면 구간을 잘못 넣기 쉽다.
- 각 판정마다 걸린 규칙 번호 + 근거 숫자
- 코드 미기재·표본 부족 캠페인은 맨 아래 따로 (판정 보류)
- 아무것도 실행하지 않고 제안만. 마지막에 "다음 액션" 제시.

자체 점검 (출력 직전 1회): 끄기 후보 칸에 넣은 캠페인이 전부 BEP×2를
넘겼는가? 하나라도 안 넘겼으면 유지로 옮긴다.

### /소재생성
카피 + 소재 설계까지 한 세트로 만든다.
입력받을 것: 상품/서비스, 가격 또는 상담단가, 구매자, 사용자, BEP.
1. 이 카테고리의 실제 소비자 리뷰·커뮤니티 글(디시·뽐뿌·네이트판 등)을
   웹에서 검색해 반복되는 불만/욕구 10개를 소비자 표현 그대로 수집.
2. 각각을 "타겟의 상황 + 제안 + 행동 유도" 공식으로 카피 20개 생성.
   4가치(기능/정서/사회/미적) 중 2개 이상 결합, 가치 조합 표시.

   **카피 6원칙 (반드시 지킬 것):**
   - (1) 통계는 1단계에서 수집한 글에 실제로 있던 숫자만 쓴다. 출처가
     없으면 숫자를 빼고 정성 표현으로 바꾼다 ("70.9%" → "10곳 중 7곳").
     기억나는 숫자를 쓰거나 지어내지 않는다. 출처 표기("○○ 조사")는
     카피 끝에도, 그림 안에도 넣지 않는다. (광고에 박힌 틀린 숫자와 검증
     안 된 출처는 심의 대상이고 계정에 흠이 남는다.)
   - (2) 경고형과 방법형을 5:5로 섞는다. 20개를 전부 공포(폐업·빚·당한다)로
     채우지 않는다. 절반은 "N가지 기준/방법"의 이득형으로 뒤집는다.
   - (3) 후킹 절반에 '리스트 약속'을 심는다. 숫자+명사로
     ("계약 전 물어야 할 3가지", "폐업 안 하는 자리 고르는 7가지 기준").
     구체적 리스트가 저장·클릭을 부른다.
   - (4) 가능하면 결과 변화를 화살표로 못 박는다
     ("권리금 1337만 → 나올 땐 422만", "월세 300 나가던 자리 → 고정비 0").
   - (5) 서브 한 줄에 자기 투사 장치를 넣는다
     ("지금 이 글 보는 사장님 얘기일 수 있습니다").
   - (6) CTA는 이득 회수형으로. "확인/보기/신청" 대신 무엇을 받는지 명시한다
     ("7가지 기준 받기", "원가표 받기", "폐점 지점 자료 받기").

3. 5항목 체크리스트(타겟 포함/제안 포함/가치 태그/고객 언어/CTA)로
   검수해 통과분만.
4. 통과 카피마다 아래를 표로 (사람이 훑어보는 용도):
   | 번호 | 후킹 | 서브 | CTA | 4가치 | 추천 포맷 |
   - 추천 포맷은 아래 8종 중 카피 감정에 맞게 택1. 같은 소구는 서로 다른
     포맷 2~3개로 분산 추천(포맷도 테스트하도록).

5. **카피 20개 중 상위 6개를 골라, 그 6개만 이미지 지시문을 코드 블록
   하나씩 따로 출력한다.** 20개를 다 뽑으면 화면이 길어지고 어차피 다
   만들지 않는다. 고르는 기준은 (1) 통증이 가장 구체적인 것 (2) 세 타겟과
   포맷이 골고루 섞이도록. 어느 번호를 왜 골랐는지 한 줄로 밝힌다.
   나머지 14개는 "번호를 말하면 그 지시문을 뽑아드립니다"로 남긴다.

   사용자는 이 블록을 통째로 복사해 그림 생성 도구에 붙여넣는다.
   표를 복사해 붙이면 글자가 뭉개져서 엉뚱한 그림이 나온다. 표는 표대로
   두고, 붙여넣을 것은 아래 형태로 완성된 문장이어야 한다.

   ```
   [1번] 뉴스 기사형
   정사각형 1:1 광고 이미지. 모바일 피드용 카드뉴스 스타일.

   전체 배경: 깨끗한 흰색.
   맨 위: 빨간 가로 띠가 화면 폭을 꽉 채운다. 흰 글씨 굵게 "⚠️ 사장님 필독".
   제목(굵은 검정 고딕, 두 줄): "본사 말만 믿었다가 / 원금도 못 뽑았습니다".
     '원금도 못 뽑았습니다'는 빨간색, '믿었다가'에 노란 형광펜 하이라이트.
   가운데: 모서리가 둥근 사진 박스. 텅 빈 가게에 지친 표정으로 앉은 40대 사장 실사.
   사진 아래: 연분홍색 둥근 박스. 검정 글씨 "폐점 지점부터 보여드립니다".
   맨 아래: 남색 둥근 버튼. 흰 글씨 "폐점 지점 자료 받기". 가장자리에서 여백을 둔다.

   지킬 것:
   - 글자는 위 덩어리가 전부다. 기사 본문 같은 잔글씨를 넣지 마라.
   - 한글 맞춤법 정확히. 글자가 깨지면 다시 그려라.
   - 실제 브랜드 로고, 실존 인물 얼굴을 넣지 마라.
   - 날짜, 기자 이름, 신문사, 출처 표기 같은 가짜 정보를 넣지 마라.
   ```

   지시문 규격 (전 포맷 공통, 반드시 지킬 것):
   - 비율은 정사각형 1:1로 고정. 세로로 길면 피드에서 잘린다.
   - 배경은 흰색이 기본. 카드뉴스처럼 정보 콘텐츠로 읽히게 한다.
     (검정 배경은 자극 광고로 읽혀 강의·브랜드에 안 맞는다.)
   - 색은 다섯 가지 안에서만: 흰 배경, 검정 본문, 빨강 포인트, 노랑 형광펜,
     남색 버튼. 이 밖의 색을 쓰지 않는다.
   - 강조는 형광펜 하이라이트로. 핵심 단어에만 노란 형광펜을 긋는다.
     글자 전체를 형광색으로 칠하지 않는다(요란해진다).
   - 글자 덩어리는 최대 3개. 후킹, 서브 한 줄, 버튼 문구.
   - 맨 위에 빨간 경고 띠("⚠️ 사장님 필독" 등)를 둘 수 있다(선택, 경고형에 어울림).
   - 사진은 모서리가 둥근 박스 안에 넣는다. 인물 나이·표정·조명까지 한 문장으로.
   - 버튼은 남색 둥근 형태로 화면 아래, 가장자리에서 떨어뜨린다(잘림 방지).
   - "기사 본문", "설명 문단", "표", "체크리스트 항목 6개" 같은 잔글씨 금지.
     화면에서 안 읽힌다.
   - 마지막에 "지킬 것" 4줄을 그대로 붙인다. 이게 없으면 그림 도구가
     가짜 기사와 잔글씨를 만들어낸다.

소재 포맷 8종 (각 포맷의 글자 덩어리 배치):
1. 정보/체크리스트형 — 항목은 **3개까지만**, 큰 글씨. 깔끔한 신뢰.
2. 비교형(잘못 vs 검증) — 좌우 반반. 각 면에 짧은 문구 하나씩.
3. 뉴스 기사형 — 큰 헤드라인 톤만 빌린다. 본문·날짜·기자명 금지.
4. 후기/인터뷰형 — 인물 사진 + 따옴표 한 문장 + 별 다섯 개.
5. 큰 숫자 임팩트형 — 숫자 하나가 화면 절반. "70%".
6. 카톡 상담 대화형 — 말풍선 **2개까지**. 긴 대화 금지.
7. 감성 상황 사진형 — 사진이 주인공. 글자는 후킹 한 줄만.
8. 손글씨 경고 메모형 — 포스트잇에 손글씨 한 문장.

주의: 이미지 생성 자체는 이 스킬 밖(ChatGPT 등)에서 한다. 여기서는
붙여넣을 지시문까지 완성해서 넘긴다.

### /채점
소재나 광고를 점수화한다.

**소재를 어디서 가져오나 (계정이 연결돼 있을 때)**
- 광고 문구(본문·헤드라인·버튼 종류)는 계정에서 직접 조회한다. 사용자가
  붙여주기를 기다리지 않는다.
- **이미지 그림 자체는 도구로 못 받아온다.** 페이스북 이미지 서버가 막혀
  있다. 문구만 보고 채점을 끝내지 말고, "광고 관리자 미리보기 화면을
  캡처해서 올려달라"고 요청한다. 캡처가 오면 그때 이미지 항목까지 채점.
- 캡처를 못 받으면 이미지 항목은 "확인 불가"로 두고 나머지만 점수화한다.
  총점의 분모를 몇 점 만점인지 함께 밝힌다.

**채점** 항목별 0~5점:
후킹(도입 3초) / 카피(고객 언어) / 행동 유도 / 오퍼(제안 매력) /
4가치 결합 / 반응 지표(데이터 있으면). 총점 + 가장 약한 항목 **하나** +
개선안 **한 줄**. 여러 개 늘어놓지 않는다. 후킹 점수 낮으면 "집행 전
수정" 경고.

**승자와 패자를 같이 채점할 때 (가장 값어치 있는 사용법)**
성과가 갈린 두 소재를 나란히 채점한 뒤, 승자에서 **재현 가능한 골격**을
뽑아 마지막에 적는다. 다음 소재를 이 골격에 끼워 넣으면 되도록.

```
[배지]  타겟 호명
[후킹]  도발형 질문
[서브]  타겟이 실제로 쓰는 통증어 3개 나열
[오퍼]  통증의 정반대
[CTA]   저마찰 리드마그넷
```

위는 한 계정에서 나온 예시일 뿐이다. 골격은 그 계정의 승자 소재에서
직접 뽑는다. 미리 정해둔 틀에 억지로 맞추지 않는다.
승자와 패자가 왜 갈렸는지는 추측이 아니라 두 소재의 차이로만 말한다.

## 참고
- 경쟁사 광고 자동 수집은 넣지 않았다. 공식 도구가 남의 광고 캡션
  앞부분만 주고 카피 전문·이미지 글자·정확한 게재일수는 못 줘서
  "경쟁 분석"이라 부를 실효가 없다. 경쟁사는 사용자가 메타 광고
  라이브러리를 직접 열어 보고, 필요하면 그 캡처를 /채점에 넣는다.
- 메타에서 가장 중요한 건 소재다. 이 스킬은 판단·생성의 하방을 막아줄 뿐,
  터지는 소재(원천 콘텐츠)는 사람 몫이다.
