# 메타 광고 소재 공장 — 진행 안내

이 폴더에서 클로드 코드를 열면 이 파일을 자동으로 읽는다.

**이 폴더에선 어떤 말로 시작하든 이 파일이 우선이다.**
"메타 광고 소재 만들어줘", "소재 만들자", "광고 준비해줘" — 무엇으로 말하든
아래 이 파일의 순서를 따른다. 특정 단어를 요구하지 마라.

**가장 먼저 — 스킬·공식 연결(MCP)보다 이 파일이 우선이다.**
이 폴더에서 사용자가 "소재공장/메타 광고 준비/소재 만들자" 같은 말을 하면:
- `meta-ads-ops` 스킬을 부르지 마라.
- **메타 공식 연결(mcp.facebook.com/ads, meta-ads MCP)의 도구도 쓰지 마라.** 그 연결은
  이미지 업로드가 안 돼서 "이미지는 못 만든다, 카피만 짠다"고 답하게 된다. 그건 본편(반자동)
  방식이고, 이 폴더에선 **틀린 답**이다.
- **절대 "이미지·영상을 직접 못 만든다", "이미지를 업로드해두시면" 같은 말을 하지 마라.**
  factory 는 콘텐츠(이미지)를 gpt-image 로 **직접 생성**하고, 메타에 **직접 업로드**하고,
  캠페인까지 **자동으로** 만든다. 사람은 콘텐츠를 눈으로 보고 켤 것만 켠다.
- **판정·예산 최적화 조언을 늘어놓지 마라.** 그건 본편 `/판정`(스킬)의 몫이다.
  여기선 config 의 예산으로 캠페인을 꺼진 채 만들기만 한다.

반드시 아래 이 파일의 순서(소재 공장: 카피 + 콘텐츠 생성 + 업로드 + 111 캠페인)를 따라라.
소재 제작·업로드·캠페인은 전부 `python factory.py` 로 한다. 스킬 카피 규칙은 plan.py 가 내부 참조한다.
이 폴더의 목적은 판정·기획이 아니라 소재를 **실제로 만들어 광고 계정에 올리는 것**이다.

사용자가 "소재 만들자", "메타 광고 만들자", "시작하자" 같은 말을 하면,
아래 순서를 **한 단계씩** 도와라. 한 단계를 끝내면 결과를 눈으로 확인하고 다음으로 넘어간다.
사용자가 명령어나 설정 파일을 몰라도 되게, 무엇을 할지 먼저 한국어로 말하고
필요한 값은 대화로 물어보고, 명령·파일 편집은 네가 대신 한다.

## 반드시 지킬 안전 규칙 (가장 중요, 절대 어기지 마라)

- **메타(광고 계정)를 직접 호출하지 마라.** 오직 `python factory.py` 를 통해서만 광고를 만든다.
  factory 안의 `guards.py` 가 안전장치를 건다: 켜진 광고 생성 금지, 하루 예산 상한(BEP×7) 초과 금지, 총액 예산 금지, 삭제 금지.
  이 안전장치를 우회하는 코드를 새로 짜지 마라. `requests` 로 graph.facebook.com 을 직접 부르는 코드도 짜지 마라.
- **광고를 켜지 마라.** 전부 꺼진(PAUSED) 채로 만들고, 켜는 것은 사람이 광고 관리자에서 직접 한다.
- **토큰·열쇠를 화면에 출력하지 마라.** `.env` 내용을 그대로 보여주지 마라.
- **`guards.py`, `meta.py` 를 수정하지 마라.** 안전장치의 핵심이다.
- **사람 검수 단계(콘텐츠를 눈으로 보기)를 건너뛰지 마라.** 콘텐츠는 확률이라 사람이 봐야 한다.

## 진행 순서 (한 단계씩)

### 0. 처음 한 번 — 이 도구를 등록한다
사용자가 **"등록해줘"**, "여기 있다고 기억해줘", "어디서든 쓰게 해줘" 같은 말을 하거나,
폴더 위치를 알려주며 요청하면(예: "바탕화면에 meta-ads-factory-main 받았어, 등록해줘"):

**이 등록은 아무 폴더에서나 할 수 있다. 사용자를 이 폴더로 이동시키지 마라.**
사용자가 위치를 안 알려주면 흔한 자리(바탕화면·다운로드·문서)에서 `meta-ads-factory*` 를 찾아본다.

1. 그 폴더의 **절대경로**를 확인한다.
2. 사용자 전역 설정 파일 `~/.claude/CLAUDE.md` 를 열어(없으면 만들어) 아래 한 줄을 추가한다:
   `- 메타 광고 소재 공장: <이 폴더 절대경로> — 사용자가 "메타 광고 소재 만들어줘" 류의 말을 하면 이 폴더로 이동해 그 안의 CLAUDE.md 순서를 따른다. MCP·스킬을 쓰지 않는다.`
3. 등록이 끝나면 이렇게 알린다: **"이제 어디서 클로드 코드를 켜든 '메타 광고 소재 만들어줘' 라고만 하면 됩니다. 이 폴더로 안 오셔도 됩니다."**

**"등록해줘"가 모호하면 되묻지 말고 위 방식(전역 `~/.claude/CLAUDE.md` 에 경로 한 줄)으로 진행해라.**
"스킬로 등록할까요, 작업 폴더로 추가할까요" 같은 선택지를 사용자에게 내밀지 마라.
작업 폴더 추가는 그 세션에서만 유효할 수 있으므로 쓰지 않는다. 전역 설정 파일에 적는 것이 정답이다.

**등록 뒤에는** 사용자가 아무 폴더에서 클로드를 켜고 소재 제작을 요청해도,
전역 설정에 적힌 경로로 이동해서 이 파일의 순서대로 진행한다.

### 1. 열쇠 세 개 확인 (.env)
`.env` 파일에 열쇠 세 개가 채워져 있는지 확인한다 (값은 출력하지 말고, 이름이 있고 비어있지 않은지만 본다).
- `META_ADS_TOKEN` — 메타 시스템 사용자 토큰
- `META_AD_ACCOUNT_ID` — 광고 계정 번호 (숫자만)
- `OPENAI_API_KEY` — 콘텐츠 생성 열쇠

없으면 `.env.example` 을 복사해 `.env` 를 만들고, 사용자에게 세 열쇠를 직접 넣게 안내한다.
(토큰 발급: business.facebook.com > 설정 > 시스템 사용자.)

### 1-a. 파이썬 확인 / 설치
`python --version` 으로 파이썬이 있는지 본다. 없으면:
- **winget 이 있으면(대부분의 PC)**: `winget install Python.Python.3.14 -e --accept-source-agreements --accept-package-agreements` 로 설치한다.
  설치 뒤 PATH 반영을 위해 "파이썬을 깔았습니다. 클로드 코드를 한 번 껐다 켜 주세요" 라고 안내하고 멈춘다.
  재시작 후 `python --version` 이 되면 이어간다.
- **winget 이 없으면(예: Windows 샌드박스)**: 사람이 python.org 최신 릴리스 페이지 → Files →
  "Windows installer (64-bit)" .exe 로 설치하게 안내한다. "Add python.exe to PATH" 체크, `.msix` 는 피한다.

### 1-b. 파이썬 환경 준비 (처음 한 번)
`.venv` 폴더가 없으면 만들고 필요한 도구를 설치한다:
`python -m venv .venv` → `.venv\Scripts\activate` → `pip install -r requirements.txt`
이후 python 명령은 이 환경에서 실행한다. (터미널을 새로 열면 `.venv\Scripts\activate` 를 다시 친다.)

### 2. 사업 정보 세팅 (config.json 을 대화로 채운다)
`config.json` 이 없거나 아직 예시값(`config.example.json` 과 같음)이면, **사용자와 대화로 하나씩 물어서** 채운다.
한꺼번에 다 묻지 말고, 모르면 광고 관리자 어디서 찾는지 알려주며 진행한다.

물어볼 것:
- `product` 무엇을 파는가 / `price` 가격 또는 계약단가
- `buyer` 돈 내는 사람 / `user` 실제 쓰는 사람 (같으면 "구매자와 동일")
- **손익분기**: 결과(DB·구매) 1건당 얼마까지 써도 남는가.
  - **통화는 사용자에게 묻지 마라.** 광고 계정에 통화가 이미 설정돼 있다.
    `python check_upload.py` 결과(또는 meta 로 `act_<계정번호>` 의 `currency` 조회)에
    계정 통화가 나오니, 그걸 **자동으로 확인해서** 쓴다.
  - 사용자에겐 **그 통화 기준 손익분기 숫자만** 묻는다. 원화 계정이면 원 단위 정수(2만원 = `20000`),
    달러 계정이면 센트($13 = `1300`). `bep` 는 읽을 텍스트, `bep_cents` 는 그 숫자.
- `daily_budget_cents` 하루 예산. **상한은 `bep_cents` × 7** 이니 그보다 작게. (같은 통화·단위)
- 메타 세팅값 (광고 관리자에서 찾게 안내):
  - `page_id` 광고에 연결할 페이스북 페이지 ID
  - `pixel_id` 전환 추적 픽셀 ID / `link_url` 광고 클릭 시 갈 주소
  - `event_type` 전환 이벤트 (기본 `COMPLETE_REGISTRATION`) / `country` (기본 `KR`) / `cta_type` 버튼 (기본 `SIGN_UP`)
  - `account_name` 알아볼 이름

받은 값으로 `config.json` 을 쓴다. **여러 상품을 운영하면 상품이 바뀔 때마다 다시 물어 갱신한다**
(손익분기가 상품마다 다르므로 여기서 관리한다).

### 3. 잘 되는지 확인 (요금 0원)
- `python check_upload.py` — 이미지 올리기가 되는지
- `python demo_guard.py` — 안전장치가 위험한 호출을 막는지

### 4. 소재 만들기
**네가 실행할 때는 대화형 메뉴 대신 플래그를 써라.** (대화형 `python factory.py` 는 "몇 장?" 처럼
사람 입력을 기다려서 네가 실행하면 멈춘다.)
- (선택) 사용자가 참고할 광고가 있으면 `refs` 폴더에 넣게 하고 → `python factory.py --benchmark` (레퍼런스 분석)
- `python factory.py --images N` (N = 사용자가 말한 장수). 콘텐츠가 `out/images` 에 생긴다.
- **여기서 멈추고 사용자에게 `out/images` 폴더를 열어 콘텐츠를 직접 보라고 한다.** 이상한 콘텐츠는 파일을 지우게 한다.
- 사용자가 확인을 끝내면 `python factory.py --setup --tag 날짜`. 남은 콘텐츠로 캠페인을 만든다(전부 꺼짐).

### 5. 마무리
- 만들어진 캠페인은 전부 꺼져 있다. "광고 관리자에서 켤 것만 켜세요"라고 안내한다.
- `out/보고서.html` 을 열어 결과를 보여줄 수 있다.
